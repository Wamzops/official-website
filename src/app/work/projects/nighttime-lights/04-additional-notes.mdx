## Key Decisions

- Decide early whether the main deliverable is **(a) provincial GDP inference** (most defensible given labels) or **(b) higher-resolution GDP mapping** (harder to validate).
- Explicitly document how load shedding and lighting technology changes will be treated (covariate, normalization, or robustness checks).
- Keep a clear separation between:
  - **ETL** — reproducible GIS processing
  - **Modeling** — baselines → ML
  - **Communication** — maps, uncertainty, caveats

## Candidate Auxiliary Datasets for South Africa

- **Population density:** GRID3, Meta HRSL, Kontur
- **Built-up / land use:** Dynamic World / GHSL
- **Roads/POIs:** OpenStreetMap

## Reproducibility Notes

To rerun this project safely:

1. Install the Python packages used in the notebook.
2. Configure AWS credentials outside the notebook.
3. Ensure the S3 bucket paths still exist and are accessible.
4. Re-run raster trimming before the modelling cells if the `SA_trimmed` outputs are not already present.
5. Rebuild `df_ntls`, reload the indicator dataset, and then rerun the modelling experiments.
