## Interpretation of Results
Several patterns stand out:

- **Nighttime lights are informative.** Even the simpler ridge and lasso baselines explain a substantial share of GDP variation.
- **Non-linear models help.** XGBoost and EBM outperform the linear baselines on the nighttime-light-only feature set, suggesting that the mapping from regional light intensity to GDP is not purely linear.
- **Merged features dominate.** Combining nighttime lights with macro indicators produces extremely high `R^2` values, which probably means the merged dataset contains strong predictive information, but also that the evaluation may be too optimistic for such a small annual sample.
- **Interpretability matters.** The EBM result is especially interesting because it offers strong performance without giving up inspectability to the same degree as XGBoost.

## Methodological Caveats
This notebook is promising, but the README should also document its limitations clearly:

- **Very small sample size.** Annual data from 1992-2022 gives only about 31 observations after reshaping, which is tiny for the number of features being considered.
- **Potential temporal leakage.** Random train/test splits on time-indexed macroeconomic data can overstate performance if later-year structure leaks into training.
- **Overfitting risk.** The merged dataset contains many predictors relative to sample size, which can produce inflated `R^2` on a random holdout.
- **Unclear target granularity.** The response is national GDP, while the nighttime-light features are provincial totals. That is workable, but it should be framed explicitly as national GDP predicted from subnational spatial aggregates.
- **Reproducibility issue.** The notebook currently contains hard-coded AWS credentials. Those should never be published and should be replaced by environment variables or a secrets manager.
- **Placeholder sections remain.** Several cells reference code to be added from other notebooks, so the project appears partially consolidated rather than fully finalized.

## Recommended Improvements
For extended beyond exploratory analysis, the next high-value steps will be:

1. Replace random train/test splits with **time-aware validation** such as rolling-origin or blocked time splits.
2. Report additional metrics such as **MAPE**, **MAE**, and confidence intervals across multiple splits.
3. Test alternative raster summaries such as mean, median, lit-area share, or top-decile intensity, not just summed brightness.
4. Add feature-importance or shape-function visualizations, especially for EBM and XGBoost.

## Reproducibility Notes
To rerun this project safely:

1. Install the Python packages used in the notebook.
2. Configure AWS credentials outside the notebook.
3. Ensure the S3 bucket paths still exist and are accessible.
4. Re-run raster trimming before the modelling cells if the `SA_trimmed` outputs are not already present.
5. Rebuild `df_ntls`, reload the indicator dataset, and then rerun the modelling experiments.
